Lean 4 for Economics and Computation

Machine-checked formalizations of EC research papers.

EconCSLib combines reusable Lean infrastructure with paper-specific validation artifacts: theorem interfaces, source-aligned status ledgers, dependency graphs, and human-readable audit reports.

Project

What the repository is for

The reusable library under EconCSLib/ collects foundations for probability, optimization, matching, auctions, online algorithms, recommender systems, and fair division.

The paper folders under papers/ preserve a source paper's notation, theorem numbering, proof modules, validation report, review dashboard configuration, and compact PaperInterface.lean file.

Current verification status

Published paper artifacts

This site is a short human-facing snapshot. The source of truth is the repository status ledger and the paper-local validation reports.

Verified or formalized

Public paper artifacts cover matching, digital-goods auctions, online advertising, recommender-system fairness, test-optional admissions, driver surge pricing, Bayesian rating design, and discretization-induced bias.

Public partial

LMMS04 fair division is public because its remaining gap is an explicit reusable fixed-dimension integer-program runtime seam, represented in Lean as ExternalSolverConsequence.

Contribution path

New paper formalizations can be developed privately, then proposed through a pull request once the theorem interface, validation report, and caveats are ready for public review.

Architecture

Library code plus reviewable paper trails

Reusable library

General definitions and lemmas live in the shared EC library when they are likely to support more than one paper.

Paper folders

Paper-specific files keep source-faithful formal statements, proof code, theorem ledgers, dependency graphs, and validation notes together.

Agent workflow

The public formalizer skill bundle records the repeatable process for ingesting a paper, translating claims, proving them in Lean, and surfacing human-review seams.
