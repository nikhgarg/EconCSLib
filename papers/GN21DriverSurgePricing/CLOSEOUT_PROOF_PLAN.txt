Meaning of r1_current
---------------------
r1_current is the measured one-state reward rate of the current fixed
non-surge policy:

  gn21MeasuredStateRewardRate
    (μ 0) (arrival 0)
    (ctmcStructuredDynamicSurgePrice m z switch12 switch21 0)
    (ρ 0)

It is not a raw expected payment.  It is W_1(σ_1) / T_1(σ_1), where W_1 is
average payment per accepted trip and T_1 is the state cycle time.

The paper's Lemma 9 fixes arbitrary σ_1, and thus Q_1, T_1, R_1.  In the
Theorem 3 proof that fixed-state reward rate is this current policy-dependent
r1_current, not the target accept-all R1 unless σ_1 is already accept-all or a
separate equality has been proved.


What is easy versus not easy
----------------------------
Nonnegativity of r1_current should be easy once the current non-surge payment
is pointwise nonnegative on the accepted policy:

  accepted payment integral >= 0
  accepted mass > 0
  state cycle time > 0
  therefore measured reward rate >= 0

This has now been started in Lean with generic lemmas around
singleStateTripPayment_nonneg_of_pointwise_nonneg,
gn21StateCycleTime_pos_of_mass_pos, and
gn21MeasuredStateRewardRate_nonneg_of_pointwise_payment_nonneg.

The upper bound

  r1_current <= R1

does not follow from nonnegative payments or from integrating over a larger
accepted set.  Adding nonnegative trips can lower or raise a rate, because the
denominator changes.  This upper bound must come from a rate comparison:
accept-all is at least as good as the current non-surge policy for the
constructed non-surge price, or from the equivalent rejected-complement
average-rate comparison.

Step 2: prove pointwise nonnegativity of the constructed non-surge price.

The difficult-looking issue is that Theorem 3 allows z_1 to be negative.  This
is still manageable from the Lemma 10 lower bound.

For the non-surge price,

  w_1(τ) = R2 * τ + z_1 * q_{1->2}(τ)
         = R2 * (τ + ratio * q_{1->2}(τ))

where z_1 = ratio * R2 and R2 > 0.

If ratio >= 0, use q >= 0 and τ > 0.

If ratio < 0, use q_{1->2}(τ) <= switch12 * τ.  It is enough to show

  1 + ratio * switch12 >= 0.

The Lemma 10 lower bound gives

  ratio > -(T2*switch12 + Q2)
            / (Q2*(switch12*T1 - Q1) + switch12*(T2*switch12 + Q2)).

Because Q2 > 0 and switch12*T1 - Q1 > 0 for the accept-all non-surge
primitive, the denominator is strictly larger than

  switch12 * (T2*switch12 + Q2).

Therefore the lower bound is strictly greater than -1/switch12.  Hence

  ratio > -1/switch12
  1 + ratio*switch12 > 0

and pointwise w_1(τ) >= 0 follows for all positive τ.

Step 3: prove r1_current <= R1 as a rate comparison, not an integral monotonicity fact.

Target shape:

  gn21MeasuredStateRewardRate current_nonsurge_price (ρ 0)
    <=
  gn21MeasuredStateRewardRate current_nonsurge_price acceptAllPolicy

Then rewrite the accept-all reward-rate using Theorem 3 accounting:

  gn21MeasuredStateRewardRate ... acceptAllPolicy = R1.

There are two possible routes.

Route A, likely fastest:
Use the already compiled Lemma 10/non-surge-after-surge aggregate branch.  It
already proves that, with surge fixed at accept-all, moving non-surge to
accept-all weakly improves aggregate reward.  Since the surge state and its
time/reward terms are fixed in that branch, extract or add a one-state
consequence that the non-surge current rate is <= the target accept-all rate.

This avoids reproving Lemma 10 derivative positivity.

Route B, more direct:
Use the single-state accepted/rejected-set algebra for the non-surge price.
Show that every rejected complement piece has average rate at most R1, then
adding it cannot reduce the rate comparison in the wrong direction.  This is
probably more measure/integral work than Route A, but it is conceptually
simple.

Do not try to get r1_current <= R1 from raw payment monotonicity.  That is
false for rates.


Step 4: prove the current Lemma 9 lower endpoint directly.

Step 5: prove the remaining upper fixed-state cross-ratio comparison.

The current frontier still asks for:

  acceptAll_T1 * current_Q1 <= current_T1 * acceptAll_Q1

This says the current non-surge exit-weight/time ratio is at least the
accept-all ratio.  It is exactly the fixed-state upper transfer needed for the
surge Lemma 9 route.

Use the complement pointwise comparison if possible:

  current_Q1 * τ >= current_T1 * q_{1->2}(τ)

on rejected non-surge trips, or its already compiled helper in the
gn21FixedStateCross_* family.  This comes from the fact that q(τ)/τ is
decreasing in τ and the rejected complement lies on the appropriate side of
the current non-surge threshold/shape.

If the current policy shape is not yet available at this boundary, route
through the existing Theorem 4 allowed-policy-form endpoint packages rather
than making an unstructured arbitrary-policy claim.
