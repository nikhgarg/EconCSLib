## Intended proof route

The clean human-audit route should be:

1. Prove or supply a clock-sorted, no-duplicate finite exact-drop schedule.
2. Build an exact schedule with
   `paper_theorem8_bstar_ranked_threshold_exact_drop_schedule_of_clock_sorted_nodup`.
3. Convert it to a local annotated no-overshoot history with
   `paper_theorem8_bstar_ranked_threshold_exact_drop_schedule_to_no_overshoot_strategy_history`.
4. Build a no-overshoot terminal-history certificate with
   `paper_theorem8_bstar_ranked_threshold_no_overshoot_terminal_history_behavior_certificate_of_clock_sorted_nodup_unscheduled_threshold_gt`.
   For a nonempty schedule, use the `...last_threshold_lt_unscheduled` endpoint
   to compare unscheduled thresholds to the last scheduled threshold instead of
   unfolding the recursive final-state clock. From the canonical cold start,
   use the `...cold_start_threshold_sorted...`, `...cold_start_singleton...`,
   or `...cold_start_pair...` no-overshoot terminal-dynamic endpoints when the
   schedule is a small displayed list; these discharge initial activity and
   reduce sortedness to local threshold checks.
5. Use the no-overshoot terminal-history exact-record/outcome endpoints. For
   the terminal-record local-deviation game, prefer the completed-rank
   `...completed_threshold_le` endpoints when checking a finite paper instance;
   they avoid an all-ranks terminal-inactivity premise.
   If the instance starts from the canonical no-dropout state and the scheduled
   list is nonempty, threshold-sorted, and no-duplicate, use the cold-start
   `...threshold_sorted_nodup_last_threshold_lt_unscheduled` terminal-record
   game endpoints directly. For one- or two-rank displayed schedules, prefer
   the `...cold_start_singleton...` or `...cold_start_pair...` terminal-record
   game endpoints; they expose the smallest local threshold premises.
   If the review target is the paper's unique-PBE statement, use the
   `...exists_unique_pbe_with_completed_rank_conclusion...` wrappers instead
   of manually combining `PBE iff named strategy` with rank-local conclusions;
   there are direct cold-start singleton and pair forms for small schedules.
6. If source semantics are needed, forget the no-overshoot certificate into the
   ordinary source-completion stack or use the direct no-overshoot
   ex-post/local-deviation source constructors.

The point is to avoid a global premise of the form "for every state and rank,
if the strategy would drop then the clock has not overshot." A finite history
only needs no-overshoot evidence on actual dropout transitions.

## Current assessment

The paper is not marked done. Theorem 8 is close at the algebra/certificate
plumbing level, and the terminal-record local-deviation game now closes the
outcome-map gap for exact/no-overshoot terminal histories whose terminal state
has every rank inactive. For sorted finite schedules, the source-shaped
terminal-record route now generates the no-overshoot terminal history and exact
records directly, and the source-extensive terminal-record route makes that
history and terminality part of the PBE predicate. The remaining hard work is
the fully general source proof: real generalized-English belief consistency,
source sequential rationality for arbitrary reachable/off-path states, and the
no-overshoot timing argument needed to turn ordinary source histories into exact
terminal records.
