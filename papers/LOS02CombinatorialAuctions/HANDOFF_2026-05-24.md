## Proof Path Forward

Do not reopen Theorem 7.2 or Theorem 10.2 unless a human reviewer finds a
statement mismatch. The concrete endpoints are already closed on the intended
source domain.

The only substantive way to move LOS02 from `Partially formalized` to
`Formalized` is to add a native computational-complexity library layer:

1. Define a machine/algorithm model for deterministic polynomial time,
   randomized expected polynomial time, and languages.
2. Define semantic classes `P`, `NP`, `RP`, `co-RP`, `ZPP`, and `co-NP`, then
   connect the existing abstract `RandomizedComplexityClassModel` fields to
   those semantics.
3. Define polynomial-time many-one reductions with a concrete runtime notion,
   then instantiate the compiled graph/set-packing/single-minded maps as
   polynomial-time reductions.
4. Formalize or import source hardness facts: Karp clique-to-set-packing
   NP-hardness and Hastad-style clique inapproximability under `NP != ZPP`.
5. Replace the external-consequence hypotheses in the Theorem 6.1 wrappers by
   native class-membership/hardness theorems.

This is a new library project, not a remaining LOS02 auction-proof gap.
